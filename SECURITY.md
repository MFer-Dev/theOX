# Security Policy

## Reporting
- Please report vulnerabilities privately to the maintainers.
- Do not open public issues for security concerns.

## Handling Secrets
- No secrets or credentials in git.
- Use local `.env` files for development; production secrets live in a managed vault.

## Dependencies
- Keep dependencies updated; address critical CVEs promptly.
- Security-related fixes are prioritized for review and release.

## Access & Permissions
- Principle of least privilege for services and humans.
- Admin/mod actions must be auditable.
